# mi: ConnectRPC + Annotation-Driven REST + iamd (sigv4any) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the `mi` service's HTTP RPC surface from Twirp to ConnectRPC, expose the same operations as RESTful HTTP via `google.api.http` annotations transcoded by `connectrpc.com/vanguard`, and require every mi RPC call on the public HTTP listener to be signed (SigV4 or SigV4A) and verified against `iamd` through `web/middleware/sigv4any` composed from the two remote `iamsts` verifiers.

**Architecture:** mi keeps its current service implementations (`switchtracker`, `events`) but gains a thin Connect adapter layer that wraps them in generated `miv1connect` handlers and a Vanguard transcoder that derives REST routes from proto annotations. Existing Twirp mounts and old path shims stay during a drain period; native gRPC on `:8081` stays untouched for in-cluster clients. Authentication is enforced by a single `sigv4any.Middleware` placed outside the transcoder so signatures are checked against the original request bytes before any protocol rewriting; `sigv4any` is generalized from concrete verifier fields to a middleware interface so it can compose the remote `sigv4/iamsts.Verifier` and `sigv4a/iamsts.Verifier`. mi holds only its own IAM credential (for signing its `GetSigningKey`/`GetPublicKey` calls to iamd) and never stores caller secrets.

**Tech Stack:** Go, buf v2 (`buf.gen.yaml` managed mode), `connectrpc.com/connect` v1.18.1 (already a dep), `connectrpc.com/vanguard` v0.4.0 (new, alpha), `google.api.http` annotations from `buf.build/googleapis/googleapis`, `web/middleware/sigv4{,a}/iamsts` (existing), `github.com/twitchtv/twirp` (kept globally), GORM/SQLite (mi), Kubernetes (alrest cluster).

## Decisions Captured (from brainstorming)

1. **Auth scope:** every mi RPC operation served on the public HTTP listener (`:8080`) requires valid SigV4 or SigV4A signatures. No public reads. This applies uniformly across Connect canonical paths, REST transcoded paths, and the existing Twirp paths during the drain.
2. **`/mcp`:** stays outside SigV4. It is not an RPC route; it is protected by the existing ingress (Basic Auth). Future work may add OIDC bearer tokens or dynamic OAuth client registration against `idp.xeserv.us`.
3. **Native gRPC on `:8081`:** stays as-is and unauthenticated, restricted to in-cluster clients via network policy. `cmd/x/cmd/mi/mi.go` keeps using it until a separate client-migration change.
4. **REST transcoder:** Vanguard, on the strength of the spike (see Appendix A). Alpha status accepted; mitigation is a single `newServer` constructor and route/response characterization tests so Vanguard can be ripped out without touching service logic.
5. **Twirp globally:** kept. `buf.gen.yaml` keeps the `protoc-gen-twirp` plugin and `github.com/twitchtv/twirp` stays in `go.mod` because other services (`iam`, `mimi`, `sanguisuga`, `xedn`, `patchouli`) still use it. Only mi's Twirp mounts are removed during drain.
6. **Credential scopes:** incoming mi requests are signed for region `us-east-1`, service `mi`. mi's own calls to iamd are signed for service `iam` (region `us-east-1`). Both algorithms are accepted on both surfaces.
7. **Drain:** Twirp mounts on mi are removed only after protocol metrics show zero successful Twirp calls for an agreed window (default 30 days).

## Global Constraints

- `internal.HandleStartup()` calls `flag.Parse()` — `cmd/mi/main.go` must not call `flag.Parse()` itself.
- Generated files under `gen/` are regenerated by `npm run generate`, never hand-edited.
- Never log signing key bytes, secret access keys, full Authorization headers, or ECDSA signatures. Log only `access_key_id`, `principal_id`, algorithm, and result.
- `sigv4any` changes must not break `cmd/iamd` (its existing local-verifier composition) or the `sigv4*` middleware tests.
- Auth middleware must sit outside Vanguard so signatures cover the original method/path/body, not Vanguard's rewritten form.
- Native gRPC `:8081` is out of scope for auth changes; do not add SigV4 enforcement to the gRPC server in this plan.
- All commits: Conventional Commits, `--signoff`.
- Proto regeneration: `npm run generate`. Full check: `npm test`. Lint: `buf lint && go build ./...`.
- mi currently exposes a `POSSE.RefreshBlog` RPC in proto that is unimplemented and unmounted. Do not annotate, mount, or migrate it in this plan.

		pb.NewEventsServer(svc.es).ServeHTTP(w, r)
	})

	mux.Handle("/glance", glance.New(svc.dao))
	mux.Handle("/mcp", mcp.New(svc.st, svc.es))

		if err := svc.dao.Ping(r.Context()); err != nil {

			return

		w.WriteHeader(http.StatusOK)
		fmt.Fprintln(w, "OK")
	})
	mux.Handle("/metrics", promhttp.Handler())
	return mux
}

func TestMIHTTPAnnotations(t *testing.T) {
	cases := []struct {
		service string
		method  string
		want    string
	}{
		{"within.website.x.mi.v1.SwitchTracker", "Members", "get:/v1/members"},
		{"within.website.x.mi.v1.SwitchTracker", "WhoIsFront", "get:/v1/front"},

	for _, c := range cases {

			svc := files.ByName(protoreflect.FullName(c.service))
			if svc == nil {

			}
			mtd := svc.Methods().ByName(c.method)
			if mtd == nil {

			rule := proto.GetExtension(mtd.Options(), annotations.E_Http).(*annotations.HttpRule)
			if rule == nil {

			}
			pat := rule.Pattern
			var got string
			switch p := pat.(type) {
			case *annotations.HttpRule_Get:

			case *annotations.HttpRule_Post:
				got = "post:" + p.Post
			case *annotations.HttpRule_Delete:
				got = "delete:" + p.Delete
			default:
				t.Fatalf("unsupported pattern %T", pat)
			}
			if got != c.want {
				t.Errorf("%s/%s: got %s, want %s", c.service, c.method, got, c.want)
			}
		})
	}
}

```go
package api

import (
	"errors"
	"fmt"

	"connectrpc.com/connect"
)

// ServiceError carries a stable Connect code plus a safe, user-facing message.
// Internal details (DAO error text, stack) are wrapped via Unwrap for logging
// but never serialized to clients.
type ServiceError struct {
	Code    connect.Code
	Message string
	Cause   error
}

func (e *ServiceError) Error() string {
	if e.Cause != nil {
		return fmt.Sprintf("%s: %s: %v", e.Code, e.Message, e.Cause)
	}
	return fmt.Sprintf("%s: %s", e.Code, e.Message)
}

func (e *ServiceError) Unwrap() error { return e.Cause }

func NewServiceError(code connect.Code, msg string, cause error) *ServiceError {
	return &ServiceError{Code: code, Message: msg, Cause: cause}
}

// Sentinels for the common cases. Use errors.As to extract *ServiceError in
// the adapter; anything that does not unwrap to a *ServiceError maps to
// CodeInternal.
var (
	ErrInvalidArgument = errors.New("invalid argument")
	ErrNotFound        = errors.New("not found")
	ErrCantSwitchToSelf = errors.New("cannot switch to yourself")
)

// MapError converts any error returned by a service method into a
// *connect.Error. Unknown errors become CodeInternal with a generic message;
// the original is logged by the caller.
func MapError(err error) error {
	if err == nil {
		return nil
	}
	var se *ServiceError
	if errors.As(err, &se) {
		return connect.NewError(se.Code, errors.New(se.Message))
	}
	if errors.Is(err, ErrInvalidArgument) {
		return connect.NewError(connect.CodeInvalidArgument, err)
	}
	if errors.Is(err, ErrNotFound) {
		return connect.NewError(connect.CodeNotFound, err)
	}
	if errors.Is(err, ErrCantSwitchToSelf) {
		return connect.NewError(connect.CodeInvalidArgument, err)
	}
	return connect.NewError(connect.CodeInternal, errors.New("internal error"))
}
```

type authConfig struct {
	iamdEndpoint string
	region       string
	incomingSvc  string
	outgoingSvc  string // "iam"
	accessKey    string
	secret       string
	maxBody      int64
}

// newAuthVerifier builds the sigv4any verifier that authenticates incoming
// mi requests by fetching verification material from iamd. mi holds only its
// own IAM credential; caller secrets never enter this process.
func newAuthVerifier(cfg authConfig) (*sigv4any.Verifier, error) {
	ua := useragent.Transport("within.website/x/cmd/mi", "https://xeiaso.net/contact", http.DefaultTransport)
	rt, err := sigv4aclient.NewSigV4ARoundTripper(&sigv4aclient.Config{
		Region:      cfg.region,
		AccessKey:   cfg.accessKey,
		SecretKey:   cfg.secret,
		ServiceName: cfg.outgoingSvc,
	}, ua)
	if err != nil {
		return nil, err
	}
	iamdClient := &http.Client{Transport: rt}

	v4 := v4iamsts.New(v4iamsts.Config{
		BaseURL:    cfg.iamdEndpoint,
		HTTPClient: iamdClient,
		Region:     cfg.region,
		Service:    cfg.incomingSvc,
		MaxBodySize: cfg.maxBody,
	})
	v4a := v4aiamsts.New(v4aiamsts.Config{
		BaseURL:    cfg.iamdEndpoint,
		HTTPClient: iamdClient,
		Region:     cfg.region,
		Service:    cfg.incomingSvc,
		MaxBodySize: cfg.maxBody,
	})

	return &sigv4any.Verifier{
		V4:  v4,
		V4A: v4a,
		Observe: func(algo string) {
			// Wired to a counter in Task 7.
		},
	}, nil
}

func loadSecret(path string) (string, error) {
	b, err := os.ReadFile(path)
	if err != nil {
		return "", err
	}
	// Trim a single trailing newline that secrets files commonly have.
	for len(b) > 0 && (b[len(b)-1] == '\n' || b[len(b)-1] == '\r') {
		b = b[:len(b)-1]
	}
